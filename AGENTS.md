# 南网以太网103协议库开发计划

> 本项目定位：**南网以太网103协议解析与通信库**
> 
> 协议本质：**IEC 104 的 APCI + IEC 103 的 ASDU**

---

## 一、协议核心差异

### 1.1 与标准IEC 103的关键差异

| 特性 | 标准 IEC 103 | 南网以太网103 |
|------|-------------|--------------|
| **传输层** | 串口 RS485 | TCP/IP（端口2404） |
| **APCI** | 无 | **4字节控制域**（参照104） |
| **帧格式** | FT1.2固定/可变帧 | 仅可变帧（68H开头） |
| **长度域** | 1字节 | **2字节**（最大2045） |
| **ASDU地址** | 1字节 | **2字节**（设备地址+CPU号） |
| **时标** | CP32Time2a(4字节) | **CP56Time2a(7字节)** |
| **遥测传输** | ASDU9/21 | **通用服务ASDU10** |
| **遥脉传输** | ASDU15 | **通用服务ASDU10** |
| **配置读取** | 无标准 | **通用服务ASDU21/10** |

### 1.2 APCI控制域格式

```
I格式（信息传输）- 用于传输ASDU:
┌────────────────────────────────────────────────────┐
│ 字节1: N(S)低8位                              LSB=0 │
│ 字节2: N(S)高7位 + 保留位                           │
│ 字节3: N(R)低8位                              LSB=0 │
│ 字节4: N(R)高7位 + 保留位                           │
└────────────────────────────────────────────────────┘
发送序号N(S)和接收序号N(R)用于防止报文丢失和重复

S格式（监视确认）- 用于确认I格式:
┌────────────────────────────────────────────────────┐
│ 字节1: 00000001                                    │
│ 字节2: 00000000                                    │
│ 字节3: N(R)低8位                              LSB=0 │
│ 字节4: N(R)高7位 + 保留位                           │
└────────────────────────────────────────────────────┘

U格式（控制功能）- 启动/停止/测试:
┌────────────────────────────────────────────────────┐
│ 字节1: TESTFR | STOPDT | STARTDT | 000011          │
│        确认/生效 | 确认/生效 | 确认/生效             │
│ 字节2-4: 00000000                                  │
└────────────────────────────────────────────────────┘

STARTDT(启动): 0x07  STARTDT确认: 0x0B
STOPDT(停止):  0x13  STOPDT确认:  0x23
TESTFR(测试):  0x43  TESTFR确认:  0x83
```

### 1.3 ASDU地址结构

```
┌─────────────────────────────────────────────────┐
│ 高字节 (设备地址)    │ 低字节                    │
│                      │ D8-D5: 定值区号           │
│                      │ D2-D0: CPU号              │
├─────────────────────────────────────────────────┤
│ 0x0000 = 子站本身                               │
│ 0x0001-0x00FE = 装置编号                        │
│ 0x00FF = 广播地址                               │
│ 0xFF00-0xFFFE = 定值区号+装置                   │
└─────────────────────────────────────────────────┘
```

## 二、库设计目标

### 2.1 核心定位
- 纯协议库，不包含业务逻辑
- 支持南网以太网103规范（上海许继）
- 仅实现三遥数据采集
- 不实现控制功能和特殊数据召唤

### 2.2 设计原则
- **分层架构**：传输层 → APCI层 → ASDU层 → API层
- **序号管理**：实现I格式序号机制
- **通用服务**：重点实现ASDU10/21编解码
- **易集成**：提供简洁的回调接口

```cpp
class IDataHandler {
public:
    virtual ~IDataHandler() = default;

    // 遥信（双点）- 总召唤响应ASDU42或突发ASDU1
    virtual void onDoublePoint(quint16 deviceAddr, quint16 infoAddr,
                               DoublePointValue value, const Quality& q,
                               const QDateTime& time) {}

    // 遥信（单点）- 突发ASDU1
    virtual void onSinglePoint(quint16 deviceAddr, quint16 infoAddr,

    virtual void onAnalogValue(quint16 deviceAddr, quint8 group, quint8 entry,
                               float value, const Quality& q,
                               const QString& unit) {}

    // 遥脉（通用服务ASDU10）- 积分总量
    virtual void onIntegratedTotal(quint16 deviceAddr, quint8 group, quint8 entry,
                                   quint32 value, const Quality& q,
                                   const QDateTime& time) {}

    // 通用服务数据（原始格式）
    virtual void onGenericData(quint16 deviceAddr, const GenericDataItem& item) {}

    // 链路状态
    virtual void onLinkStateChanged(LinkState state) {}
    
    // 总召唤完成
    virtual void onGICompleted(quint16 deviceAddr) {}
};

## 六、帧格式定义

### 6.1 APDU结构

```
┌──────┬──────────┬──────────┬─────────────────────────┬──────┐
│ 68H  │ 长度L    │ 长度L    │ APCI (4字节) + ASDU     │ CS   │
│      │ 低字节   │ 高字节   │                         │      │
└──────┴──────────┴──────────┴─────────────────────────┴──────┘

长度L = APCI长度(4) + ASDU长度
最大长度 = 2045 (APDU_MAX = 2048 - 启动字符 - 长度域(2字节))
```

### 6.2 ASDU结构

```
┌──────────┬──────────┬──────────┬──────────┬──────────────────┐
│ TI       │ VSQ      │ COT      │ ASDU地址 │ 信息对象         │
│ (1字节)  │ (1字节)  │ (2字节)  │ (2字节)  │ (可变)           │
└──────────┴──────────┴──────────┴──────────┴──────────────────┘

TI: 类型标识
VSQ: 可变结构限定词 (SQ位 + 信息对象数目)
COT: 传输原因 (2字节，高字节为PN+T)
```

### 6.3 主要ASDU类型（本项目实现范围）

> **注意**：本项目仅实现三遥数据采集，不实现控制功能

| ASDU | TI | 方向 | 用途 | 实现状态 |
|------|-----|------|------|----------|
| **遥信相关** |||||
| ASDU1 | 01H | 监视 | 带时标的报文（突发上送，含DPI） | ✓ 需实现 |
| ASDU2 | 02H | 监视 | 带相对时间的时标报文（保护动作事件） | ✓ 需实现 |
| **ASDU42** | **2AH** | 监视 | **双点信息状态（总召唤响应，南网扩展）** | ✓ 需实现 |
| **遥测/遥脉相关** |||||
| ASDU10 | 0AH | 监视 | 通用分类数据（遥测/遥脉上送） | ✓ 需实现 |
| ASDU21 | 15H | 控制 | 通用分类读命令（召唤遥测/遥脉） | ✓ 需实现 |
| **总召唤相关** |||||
| ASDU7 | 07H | 控制 | 总召唤命令 | ✓ 需实现 |
| ASDU8 | 08H | 监视 | 总召唤结束 | ✓ 需实现 |

**不实现的ASDU**（超出项目范围）：
- ASDU6 (时钟同步) - 控制功能
- ASDU12 (录波简报) - 特殊数据
- ASDU13-16 (录波文件) - 特殊数据
- ASDU20 (复归命令) - 控制功能
- ASDU17-18 (历史信息) - 特殊数据
- ASDU101-108 (通用文件) - 特殊数据

## 七、传输原因(COT)定义

### 监视方向

| COT | 含义 | 说明 |
|-----|------|------|
| 1 | 自发(突发) | 事件触发上送 |
| 2 | 循环 | 周期性上送 |
| 3 | 复位帧计数位(FCB) | 复位后首次上送 |
| 4 | 复位通信单元(CU) | 复位后首次上送 |
| 5 | 启动/重新启动 | 装置启动 |
| 6 | 电源合上 | 装置上电 |
| 7 | 测试模式 | 测试状态 |
| 8 | 时间同步 | 时间同步响应 |
| **9** | **总召唤** | **GI响应数据** |
| **10** | **总召唤终止** | **GI结束标志** |
| 11 | 当地操作 | 本地操作触发 |
| 12 | 远方操作 | 远方控制触发 |

## 十一、关键信息元素定义

### 11.1 VSQ - 可变结构限定词

```
┌─────────────────────────────────────────────────────────┐
│ D7 (SQ位):                                               │
│   0 = 信息体地址寻址顺序的信息元素                       │
│   1 = 寻址单个信息元素或综合信息元素                     │
│ D6-D0 (数目): 信息对象数目 (0-127)                       │
└─────────────────────────────────────────────────────────┘
```

### 11.2 DPI - 双点信息

```
┌─────────────────────────────────────────────────────────┐
│ D1-D0:                                                   │
│   00 = 未用/不确定                                       │
│   01 = 分/开 (OFF)                                       │
│   10 = 合/关 (ON)                                        │
│   11 = 未用/不确定                                       │
└─────────────────────────────────────────────────────────┘
```

**南网规范DPI含义**：

| 信息类型 | DPI=01 | DPI=10 | DPI=00/11 |
|----------|--------|--------|-----------|
| 开关量 | 开/OFF | 合/ON | 不确定 |
| 保护动作信号 | 动作复归/OFF | 动作出口/ON | 不确定 |
| 压板状态 | 未投入/OFF | 投入/ON | 不确定 |
| 自检信号/其他事件 | 告警复归/OFF | 告警产生/ON | 不确定 |
| 运行状态 | 检修 | 运行 | 不确定 |

### 11.3 FPT - 故障相别及类型（南网扩展）

```
┌─────────────────────────────────────────────────────────┐
│ D7: 有效位 (1=有效, 0=无效)                              │
│ D6-D5: 保留                                              │
│ D4: 区内(1)/区外(0)故障                                  │
│ D3: 接地(1)/非接地(0)故障                                │
│ D2: C相故障                                              │
│ D1: B相故障                                              │
│ D0: A相故障                                              │
└─────────────────────────────────────────────────────────┘
```

**示例**：
- 1000,1001 = A相接地故障
- 1000,0011 = AB相间故障
- 1001,0000 = 区外故障，类型未知

### 11.4 SIN - 附加信息

```
┌─────────────────────────────────────────────────────────┐
│ 当 COT=总召唤时: SIN = 扫描序号(SCN)                     │
│ 当 COT=命令认可时: SIN = 返回信息标识符(RII)             │
│ 当 COT=其他时: SIN = 无关                                │
└─────────────────────────────────────────────────────────┘
```

### 11.5 MEA - 带品质描述词的被测值

```
┌─────────────────────────────────────────────────────────┐
│ D15 (OV): 溢出标志 (0=无溢出, 1=溢出)                    │
│ D14 (ER): 差错标志 (0=有效, 1=无效)                      │
│ D13 (RES): 备用 (常为0)                                  │
│ D12-D0: 被测值 (F13格式, -1~+1, 比例因子2^12)            │
└─────────────────────────────────────────────────────────┘
```

## 十二、南网规范扩展

> **重要说明**：南网以太网103规范在标准IEC 60870-5-103基础上进行了扩展

### 12.1 ASDU地址扩展

南网规范将ASDU地址从1字节扩展为2字节：

```
┌─────────────────────────────────────────────────────────┐
│ 高字节: 设备地址 (1-254)                                 │
│ 低字节: D7-D4=定值区号, D2-D0=CPU号                      │
├─────────────────────────────────────────────────────────┤
│ 特殊地址:                                                │
│   0x0000 = 子站本身                                      │
│   0x00FF = 广播地址                                      │
│   0xFF00-0xFFFE = 定值区号+装置                          │
└─────────────────────────────────────────────────────────┘
```

### 12.2 时标扩展

南网规范使用7字节时标CP56Time2a替代4字节CP32Time2a：

```
┌─────────────────────────────────────────────────────────┐
│ 字节1-2: 毫秒 (0-59999)                                  │
│ 字节3: D0-D5=分钟(0-59), D6=IV无效, D7=备用              │
│ 字节4: D0-D4=小时(0-23), D5=SU夏令时, D6-D7=备用         │
│ 字节5: D0-D4=日(1-31), D5-D7=星期(1-7)                   │
│ 字节6: D0-D3=月(1-12), D4-D7=备用                        │
│ 字节7: 年(0-99)                                          │
└─────────────────────────────────────────────────────────┘
```

### 12.3 通用服务数据类型扩展

南网扩展的数据类型（用于ASDU10）：

| DataType | 说明 | 用途 |
|----------|------|------|
| **213** | 带绝对时间的七字节时标报文 | 遥信/事件 |
| **214** | 带相对时间的七字节时标报文 | 保护动作事件 |
| **215** | 带相对时间七字节时标的浮点值 | 故障量（遥测） |
| **216** | 带相对时间七字节时标的整形值 | 故障量（整型） |
| **217** | 带相对时间七字节时标的字符值 | 故障量（字符） |

### 12.4 KOD扩展

南网扩展的描述类别：

| KOD | 含义 | 说明 |
|-----|------|------|
| **01H** | **从装置取实际值** | 南网扩展，立即从装置读取 |
| **67H** | 属性结构 | 南网扩展，包含完整属性 |
| **168(0xA8)H** | **从子站数据库取实际值** | 南网扩展，从子站缓存读取 |
| **169(0xA9)H** | 从子站数据库读取 | 南网扩展 |

> **重要**：读模拟量时，KOD=01H表示从装置实时读取，KOD=168(0xA8)H表示从子站数据库读取

## 十四、通用服务详解

### 14.1 通用服务结构

```
ASDU10 (通用分类数据) / ASDU21 (通用分类读命令):

┌──────┬──────┬──────┬──────────┬──────┬──────┬─────────────┐
│ TI   │ VSQ  │ COT  │ ASDU地址 │ FUN  │ INF  │ 数据内容    │
│ 0A/15│      │      │          │ FE   │ F0-F4│             │
└──────┴──────┴──────┴──────────┴──────┴──────┴─────────────┘

FUN=FEH (通用分类功能类型)
INF (本项目使用的读命令):
  F0H = 读全部组的标题
  F1H = 读一组的全部条目
  F3H = 读单个条目的目录
  F4H = 读单个条目的值
```

> **注意**：INF F9H(带确认写)、FAH(带执行写)为定值修改功能，本项目不实现

### 14.2 KOD描述类别

| KOD | 含义 | 说明 |
|-----|------|------|
| 01H | 实际值 | 从装置读取 |
| 0AH | 描述 | 名称/描述字符串 |
| 67H | 属性结构 | 南网扩展，包含完整属性 |
| A8H | 从装置读取实际值 | 南网扩展 |
| A9H | 从子站数据库读取 | 南网扩展 |

### 14.3 数据类型定义（南网扩展）

| DataType | 说明 | 用途 |
|----------|------|------|

| 3 | 无符号整数 | 整数值 |
| 5 | 有符号整数 | 有符号值 |
| 7 | R32.23 浮点数 | **遥测值** |
| 22 | 通用分类失败回答码 | 错误响应 |
| 23 | 数据结构 | 复合结构 |
| **213** | 带绝对时间的七字节时标报文 | **遥信/事件** |
| **214** | 带相对时间的七字节时标报文 | **保护动作事件** |
| **215** | 带相对时间七字节时标的浮点值 | **故障量（遥测）** |

| **217** | 带相对时间七字节时标的字符值 | **故障量（字符）** |

### 14.4 通用分类数据结构

```
┌─────────────────────────────────────────────────────┐
│ RII (返回信息标识符) - 1字节                         │
│ NGD (通用分类数据集数目) - 1字节                     │
│   ┌─ CONT: 后续位                                    │
│   └─ 数目: 本帧数据集数                              │
├─────────────────────────────────────────────────────┤
│ 数据集 (重复NGD次):                                  │
│   ┌─────────────────────────────────────────────┐   │
│   │ GIN (通用分类标识序号) - 2字节               │   │
│   │   组号 (1字节) + 条目号 (1字节)              │   │
│   ├─────────────────────────────────────────────┤   │
│   │ KOD (描述类别) - 1字节                       │   │
│   ├─────────────────────────────────────────────┤   │
│   │ GDD (通用分类数据描述) - 3字节               │   │
│   │   DataType (1字节) + DataSize (1字节)        │   │

## 二十一、使用示例

### 21.1 库的使用方式

```cpp
#include <iec103/IEC103Master.h>

class MyHandler : public IDataHandler {
public:
    // 遥信回调（双点）
    void onDoublePoint(quint16 dev, quint16 addr, DoublePointValue val,
                       const Quality& q, const QDateTime& time) override {
        qDebug() << "遥信:" << dev << addr
                 << (val == DoublePointValue::On ? "合" : "分")
                 << "时间:" << time.toString();
    }

    // 遥测回调（浮点值）
    void onAnalogValue(quint16 dev, quint8 group, quint8 entry,
                       float value, const Quality& q, const QString& unit) override {
        qDebug() << "遥测:" << dev << "组" << group << "条目" << entry
                 << "=" << value << unit;
    }

    // 遥脉回调（积分总量）
    void onIntegratedTotal(quint16 dev, quint8 group, quint8 entry,
                           quint32 value, const Quality& q,
                           const QDateTime& time) override {
        qDebug() << "遥脉:" << dev << "组" << group << "条目" << entry
                 << "=" << value << "时间:" << time.toString();
    }

    // 总召唤完成
    void onGICompleted(quint16 dev) override {
        qDebug() << "总召唤完成:" << dev;
    }
};

int main(int argc, char *argv[]) {
    QCoreApplication app(argc, argv);

    IEC103Master master;
    MyHandler handler;

    master.setDataHandler(&handler);
    master.setConfig({"192.168.1.100", 2404});
    master.connectToServer();

    // 连接成功后执行总召唤（遥信）
    QObject::connect(&master, &IEC103Master::connected, [&]() {
        master.generalInterrogation(0);  // 召唤子站遥信
        master.readGenericGroup(1, 0x08);  // 召唤模拟量组（遥测）
    });

    return app.exec();
}
```
